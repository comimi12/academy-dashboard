# academy-dashboard — Claude Code 협업 가이드

이 저장소는 **SL&C 조리 아카데미 누계 대시보드**의 라이브 소스입니다.
**2명의 사용자가 각자 PC에서 교차로 업데이트**합니다. 공개 URL: https://comimi12.github.io/academy-dashboard/

## 구조 (단 2개 파일만 신경 쓰면 됨)

| 파일 | 역할 | 누가 수정 |
|------|------|-----------|
| `data.js` | 모든 숫자·월별 실적 (편집 대상) | 두 사용자 모두 |
| `index.html` | 대시보드 화면 (data.js를 동적 로드 → 항상 최신) | 거의 안 건드림 |

> 누계(합계·진행률·절감)는 화면이 자동 계산합니다. 직접 더하지 마세요.

## 월간 업데이트 절차 (Claude Code가 이대로 수행)

```bash
# 1) 항상 최신부터 — 충돌 방지 (필수)
git pull --rebase

# 2) data.js 의 monthly[] 배열 맨 아래에 그 달 실적 한 줄 추가
#    예) { ym: "2026-07", 채용: { 고급: 4, 중급: 9, 기초: 2 }, 입교: 10, 수료: 8, 배치: 8, 재직중: 27, 절감액: 1600 },

# 3) 커밋 & 푸시 → GitHub Pages 자동 반영 (빌드 불필요)
git add data.js
git commit -m "data: 2026-07 실적 추가"
git push
```

푸시 후 1분 내 https://comimi12.github.io/academy-dashboard/ 에 자동 반영됩니다.
data.js는 매 접속마다 캐시 우회로 새로 로드되므로(`data.js?_=timestamp`) 카카오톡 인앱에서도 최신이 보입니다.

## 교차 업데이트 충돌 방지 규칙 (중요)

- 작업 **시작 전 반드시 `git pull --rebase`**.
- `push`가 거부되면(상대가 먼저 올림) `git pull --rebase` 후 다시 `push`.
- 두 사람은 보통 **다른 달(다른 줄)** 을 추가하므로 충돌은 드뭅니다. 같은 줄 충돌 시 양쪽 줄을 모두 남기면 됩니다.

## data.js 입력 항목

| 항목 | 의미 |
|------|------|
| `ym` | 연-월 ("2026-07") |
| `채용.고급/중급/기초` | 그 달 트랙별 채용 인원 |
| `입교 / 수료 / 배치` | 그 달 아카데미 입교·수료·매장 배치 인원 |
| `재직중` | 그 달 말 현재 재직 인원 (시점값) |
| `절감액` | 그 달 일용직 대비 절감액 (만원) |

목표·기준선을 바꾸려면 `data.js` 상단 `targets` / `baseline` 숫자만 수정.
