# SL&C 조리 아카데미 누계 대시보드

🔗 **공개 URL:** https://comimi12.github.io/academy-dashboard/ (로그인 불필요 · 카카오톡 공유 가능)

채용 → 아카데미 → 매장 배치 인력 파이프라인의 **월별 실적을 누계로 추적**하는 대시보드입니다.
**2명이 각자 PC에서 교차로 업데이트**합니다.

## 업데이트 방법

`data.js`의 `monthly[]` 배열에 그 달 실적을 한 줄 추가하고 `git push` 하면 끝입니다.
(누계·진행률·절감액은 자동 계산. 사이트는 1분 내 자동 갱신.)

Claude Code를 쓰면 "이번 달 실적 업데이트해줘 — 채용 고급4 중급9 기초2, 입교10, 수료8, 배치8, 재직27, 절감1600"
처럼 말하면 됩니다. 상세 절차는 `CLAUDE.md` 참고.

## 협업 충돌 방지
작업 전 항상 `git pull --rebase`. 두 사람은 보통 다른 달을 추가하므로 충돌은 거의 없습니다.

## 파일
- `data.js` — 누계 KPI 데이터 (편집 대상)
- `index.html` — 대시보드 화면 (data.js 동적 로드 + 9개 뷰 탭)

## 뷰(탭) 구성

사이드바에 총 9개 탭이 있습니다 (`v1`~`v9`, `index.html` 하단 `VIEWS`/`order` 참고).

| 탭 | 내용 |
|----|------|
| 1~8 | 채용 계획·매장 시뮬레이션·기존 현황·상세 커리큘럼(개요)·인력구성·사후관리·이슈·타브랜드 벤치마킹 |
| **9. 4기 실제 커리큘럼** | 고명외식고 4기(4명) **실제** 2주 아카데미 시간표 + 현장실습(OJT) 일정 + 교육생 배치 현황 |

**뷰9는 data.js가 아니라 `index.html`에 정적 HTML로 직접 박혀 있습니다** (날짜·시간표가 자주 바뀌는 데이터가 아니라서). "오늘" 칸 강조만 파일 맨 아래 별도 `<script>`가 열람 시점 기준으로 매번 계산합니다.
- 4기 커리큘럼 내용이 바뀌면 `index.html`의 `#v9` 섹션을 직접 수정하세요 (원본 데이터는 `10-projects/11-academy-4gi-curriculum-dashboard/dashboard-app/data/*.json`).
- **교육생 주소는 의도적으로 뺐습니다** — 이 사이트가 공개(public) GitHub Pages라 개인정보 노출 위험 때문입니다. 이름·트랙·배치예정매장까지만 표시합니다.
